# 依赖管理（Dependency Management）

## 一、学习目标

完成本课后，学习者应能够：
1. 理解依赖管理在项目执行中的重要性。
2. 掌握识别和追踪依赖关系的方法。
3. 能够制定依赖缓解策略，避免项目瓶颈。
4. 熟悉工具与实践，用于持续监控依赖状态。

---

## 二、核心概念

依赖管理是识别、分析和协调项目中各任务或团队之间依赖关系的过程，确保项目顺利推进。

### 核心概念

1. **依赖类型**
    - **内部依赖（Internal Dependencies）**：项目内部任务或团队之间的相互依赖。
    - **外部依赖（External Dependencies）**：依赖外部团队、供应商或第三方资源。

2. **依赖关系管理（Dependency Management）**
    - 识别任务间依赖关系，明确关键路径。
    - 规划依赖解决方案，避免延迟蔓延。

3. **关键路径（Critical Path）**
    - 决定项目最短完成时间的任务序列。
    - 关键路径上的延迟会直接影响项目交付。

4. **依赖缓解策略（Mitigation Strategies）**
    - 调整任务顺序
    - 并行开发可行模块
    - 提前获取外部资源
    - 设置缓冲时间

---

## 三、实操方法

1. **识别依赖关系**
    - 绘制任务网络图或依赖矩阵
    - 标记任务先后顺序与关键节点

2. **分析关键路径**
    - 使用甘特图或网络图（PERT/CPM）
    - 识别影响项目总时长的关键任务

3. **制定缓解计划**
    - 对高风险依赖提前规划解决方案
    - 设置任务缓冲时间，减少阻塞风险

4. **持续监控**
    - 定期在项目会议检查依赖状态
    - 更新依赖矩阵和关键路径

---

## 四、案例分析

**案例：开发一款 SaaS 产品**

- 任务 A：后端 API 开发
- 任务 B：前端界面开发（依赖任务 A）
- 任务 C：测试环境搭建
- 任务 D：用户测试（依赖任务 B、C）

**依赖分析**：
- 关键路径：A → B → D
- 风险：A 延迟会导致 B 和 D 延迟
- 缓解策略：提前启动后端架构设计；前端先开发独立模块；增加资源加速 A 完成

---

## 五、常见问题与误区

1. **忽视外部依赖**，导致项目延迟
2. **关键路径未识别**，无法有效预测项目风险
3. **依赖矩阵未更新**，阻塞信息无法传递
4. **任务顺序硬性固定**，缺乏灵活调整

---

## 六、练习与作业

1. 针对你的项目，列出所有任务及其依赖关系。
2. 绘制依赖矩阵或网络图，标出关键路径。
3. 制定至少 3 个高风险依赖的缓解策略。
4. 每周更新依赖状态，并在团队会议中报告。
