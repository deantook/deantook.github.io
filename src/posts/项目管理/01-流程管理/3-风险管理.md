# 风险管理（Risk Management）

## 一、学习目标

完成本课后，学习者应能够：
1.	理解风险管理在项目和工程管理中的核心作用。
2.	掌握识别、评估、应对和监控风险的完整流程。
3.	能够使用工具和方法制定风险缓解策略。
4.	具备在项目执行中持续监控风险的能力。



## 二、核心概念

风险管理是系统地识别、评估和处理项目或组织可能遭遇的不确定事件的过程。

核心概念
1.	风险（Risk）
•	可能发生的事件或条件，若发生会对项目目标产生正面或负面影响。
•	类型：技术风险、资源风险、市场风险、外部环境风险等。
2.	风险识别（Risk Identification）
•	系统地发现潜在风险。
•	方法：头脑风暴、历史数据分析、专家访谈、SWOT 分析。
3.	风险评估（Risk Assessment）
•	概率（Likelihood）：风险发生的可能性
•	影响（Impact）：风险发生后对项目目标的影响程度
•	风险优先级矩阵（Probability × Impact）用于排序
4.	风险应对策略（Risk Response）
•	避免（Avoid）：改变计划以消除风险
•	减轻（Mitigate）：采取措施降低风险发生概率或影响
•	转移（Transfer）：将风险转移给第三方（如保险、外包）
•	接受（Accept）：意识到风险存在，但不采取主动措施
5.	风险监控（Risk Monitoring）
•	持续跟踪风险状态
•	更新风险登记册（Risk Register）
•	根据风险变化调整应对措施



## 三、实操方法
1.	建立风险登记册（Risk Register）

	2.	识别风险
	•	与团队成员和利益相关者开会
	•	使用过去项目的经验和数据作为参考
	3.	评估风险
	•	使用 1–5 等级评估概率和影响
	•	绘制风险矩阵，确定优先级
	4.	制定应对策略
	•	针对高优先级风险制定详细计划
	•	指定负责人和截止日期
	5.	风险监控与沟通
	•	定期在团队会议更新风险状态
	•	对发生的风险进行事后分析，总结经验



## 四、案例分析

案例：新产品上线项目
- 风险识别：技术模块延迟、关键人员离职、市场竞争变化、预算超支
- 风险评估：
- 技术模块延迟：概率高、影响高 → 优先级最高
- 关键人员离职：概率中、影响高 → 优先级高
- 市场竞争变化：概率中、影响中 → 优先级中
- 预算超支：概率低、影响高 → 优先级中
- 应对策略：
- 技术模块延迟：增加开发资源、调整迭代优先级
- 关键人员离职：制定知识传承计划、招聘备用人选
- 市场竞争变化：调整产品功能策略
- 预算超支：增加预算储备



## 五、常见问题与误区

1.	只识别高风险，忽视中低风险累积影响
2.	风险登记册不更新，导致风险应对失效
3.	风险应对策略过于泛化，无法落地
4.	团队缺乏风险意识，风险事件发生才处理



## 六、练习与作业

1.	针对你的项目，列出至少 5 个潜在风险，分别评估概率和影响。
2.	制作风险登记册，分配负责人并制定应对策略。
3.	绘制风险矩阵图（Probability × Impact），标出高优先级风险。
4.	每周进行一次风险监控会议，记录风险状态变化。
