# Golang 内建容器

Go语言（Golang）虽然不像C++或Java那样提供丰富的STL容器库，但它内置了一些基础但强大的数据结构，主要包括数组、切片和映射（map）。这些容器类型在Go标准库中作为基本构建块提供。

## 1. 数组 (Array)

数组是固定长度的、相同类型的元素的集合。

```go
var arr [5]int          // 声明一个长度为5的整型数组
arr := [5]int{1, 2, 3}  // 声明并初始化
arr := [...]int{1, 2, 3, 4, 5} // 让编译器推断长度
```

特点：

- 长度固定，不可变
- 支持索引访问
- 传递数组时会复制整个数组

## 2. 切片 (Slice)

切片是动态数组，更灵活且功能更强大。

```go
s := make([]int, 0, 5)    // 创建长度为0，容量为5的切片
s := []int{1, 2, 3}       // 直接初始化
s := arr[:]               // 从数组创建切片
```

常用操作：

```go
len(s)    // 获取长度
cap(s)    // 获取容量
s = append(s, 4)  // 添加元素
s = s[1:3]        // 切片操作
```

特点：

- 动态长度
- 共享底层数组
- 更高效的内存使用
- 是Go中最常用的序列容器

## 3. 映射 (Map)

映射是键值对的无序集合。

```go
m := make(map[string]int)       // 创建空map
m := map[string]int{"a": 1, "b": 2} // 初始化
```

常用操作：

```go
m["key"] = 42          // 添加/更新
value := m["key"]      // 获取
delete(m, "key")       // 删除
_, exists := m["key"]  // 检查存在性
```

特点：

- 无序（Go 1.0+版本中map迭代顺序随机化）
- 键类型必须可比较
- 哈希表实现，平均O(1)时间复杂度

## 4. 容器相关标准库

虽然Go内置容器有限，但标准库提供了更多容器功能：

- `container/list` - 双向链表

  ```go
  l := list.New()
  l.PushBack(1)
  e := l.Front()
  ```

- `container/ring` - 环形链表

  ```go
  r := ring.New(3)
  r.Value = 1
  r = r.Next()
  ```

## 为什么Go内置容器较少？

Go语言设计哲学强调简单性和显式性，因此：

1. 内置容器足够处理大多数基础场景
2. 避免STL式的复杂性
3. 鼓励使用简单数据结构，必要时使用标准库或自己实现
4. 保持语言核心简洁

对于更复杂的需求，开发者通常会使用第三方库或自己实现特定容器。
